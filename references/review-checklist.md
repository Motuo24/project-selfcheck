# 详细审查清单

## 安全审查

- [ ] 无硬编码的密钥、Token、密码、API Key
- [ ] 无 SQL 注入风险（参数化查询）
- [ ] 无 XSS 风险（用户输入未转义即渲染）
- [ ] 无命令注入风险（用户输入拼接到 shell 命令）
- [ ] 无路径穿越风险（用户输入拼接到文件路径）
- [ ] 权限校验完整，无越权漏洞
- [ ] 敏感数据不通过明文传输/存储
- [ ] 日志不包含敏感信息（密码、Token、手机号）

## 逻辑审查

- [ ] 所有可能为 null/undefined 的变量有判空处理
- [ ] 数组访问前检查长度，避免越界
- [ ] 边界值测试（空输入、零值、极大值、极小值）
- [ ] 异步 Promise 有 catch 或 try/catch
- [ ] 异常路径不会导致状态不一致
- [ ] 循环有明确的终止条件，无死循环风险
- [ ] 递归有 base case 且深度可控
- [ ] 类型转换安全（parseInt 脏数据、JSON.parse 异常）
- [ ] 并发操作有竞态保护

## 回归审查

- [ ] 公共 API 接口签名未破坏性变更
- [ ] 函数签名变更后所有调用方已更新
- [ ] 数据结构变更后序列化/反序列化兼容
- [ ] 依赖版本变更无 breaking changes
- [ ] 配置项变更不影响现有部署

## 需求符合度

- [ ] 用户要求的所有功能点已实现
- [ ] 输入输出格式符合预期
- [ ] 错误码/错误信息符合约定
- [ ] 无遗漏的边缘情况
- [ ] 无多余功能（scope creep）

## UI/UX 还原度（如涉及）

- [ ] 布局与设计稿一致
- [ ] 间距精确（margin/padding/gap）
- [ ] 颜色值匹配（背景、文字、边框、阴影）
- [ ] 字号、行高、字重匹配
- [ ] hover/active/disabled/focus 状态完整
- [ ] 加载态、空态、错误态已覆盖
- [ ] 响应式断点正确
