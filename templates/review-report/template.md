# 审查报告模板

## 用途

用于生成代码审查、安全审计、UI审查的标准化报告。适用于所有审查场景。

## 使用边界

- 适用：提交前全面自查、代码审查、UI审查、安全审计
- 不适用：版本号决策（使用 version-bump 模板）

## 数据形状

```json
{
  "conclusion": "通过 / 有条件通过 / 不通过",
  "summary": "一句话总结",
  "issues": [
    {
      "severity": "严重 / 警告 / 建议",
      "file": "文件路径",
      "line": "行号或范围",
      "description": "问题描述",
      "risk": "风险说明",
      "suggestion": "修复建议"
    }
  ],
  "version": {
    "current": "当前版本号",
    "suggested": "建议版本号"
  },
  "change_summary": "改动简述"
}
```

## 视觉/交互限制

- 纯文本输出，不需要 HTML 渲染
- 问题按严重程度排序
- 每个 issue 之间用空行分隔

## 边界情况

- 无问题：直接输出 "✅ 自查通过，符合提交标准"
- 仅警告：结论为"有条件通过"
- 有严重问题：结论为"不通过，建议阻断提交"
