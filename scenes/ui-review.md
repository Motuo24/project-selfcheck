# UI/UX 还原度检查

## 触发条件

- 用户提到"UI 还原度""像素级""对照设计稿"
- 代码变更涉及前端界面改动
- 用户提供了设计稿描述或截图

## 用户交互方式

**必须遵守 `references/core-constraints.md` 中的「用户交互协议（双模式）」**：
- 支持 `AskUserQuestion` 工具 → 使用结构化提问
- 不支持 → 使用纯文本多轮对话，每次只抛一个议题

## 检查维度

### 1. 布局

- 元素位置是否与设计稿一致
- 间距（margin/padding/gap）是否精确
- 对齐方式是否正确（flex/grid 布局）
- 响应式断点是否覆盖关键尺寸

### 2. 颜色

- 背景色、文字色、边框色是否与设计稿匹配
- 是否有未定义的魔法色值
- 暗色模式/主题切换是否完整

### 3. 字体与排版

- 字号、行高、字重是否匹配
- 字体族是否一致
- 文字截断/换行是否正确

### 4. 交互

- hover/active/focus/disabled 状态是否完整
- 动画/过渡是否符合设计意图
- 加载态、空态、错误态是否覆盖

## 工作流

1. 从上下文提取设计稿信息（描述、截图、figma 链接等）。
2. 逐项对照代码实现。
3. 标注偏差：位置偏移量、色值差异、字号差异。
4. 按双模式协议向用户确认是否需要调整。
5. 按 `references/reporting-standards.md` 汇报。
